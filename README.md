    <article class="markdown-body entry-content" itemprop="text"><h2><a id="user-content-oceanmesh2d-precise-distance-based-two-dimensional-automated-mesh-generation-toolbox-intended-for-coast-oceanshallow-water-flow-models" class="anchor" aria-hidden="true" href="#oceanmesh2d-precise-distance-based-two-dimensional-automated-mesh-generation-toolbox-intended-for-coast-oceanshallow-water-flow-models"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>OceanMesh2D: Precise distance-based two-dimensional automated mesh generation toolbox intended for coast ocean/shallow water flow models</code></h2>
<p align="center">
  <a target="_blank" href="/krober10nd/OceanMesh2D/blob/master/nesting.png"><img src="/krober10nd/OceanMesh2D/raw/master/nesting.png" style="max-width:100%;"></a>        
</p>
OceanMesh2D is a set of open-source MATLAB functions built off approaches and ideas from many prior works that are used to generate two-dimensional (2D) unstructured meshes for coastal circulation problems. These meshes are based on a variety of feature driven geometric--topographic mesh size functions that are generated according to user-defined parameters. Mesh generation is achieved through a force-balance algorithm combined with a number of topological improvement strategies aimed at improving the worst case triangle quality. The software embeds the mesh generation process into an object-orientated framework that contains pre- and post-processing workflows that make mesh generation flexible, reproducible, and scriptable. The objective of this paper is to introduce the functionality of the software.  
<h2><a id="user-content-code-framework" class="anchor" aria-hidden="true" href="#code-framework"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>Code framework</code></h2>
<p><code>OceanMesh2D</code>  consists of four standalone classes that are called in sequence.</p>
<pre><code>OceanMesh2D::
├── geodata -- process geospatial data.
├── edgefx  -- build mesh size functions.
├── meshgen -- generate mesh based on mesh size functions and boundaries.
└── msh     -- store, write, read, inspect, and visualize meshes and their axuillary components for numerical simulation.
</code></pre>
<h2><a id="user-content-starting-out" class="anchor" aria-hidden="true" href="#starting-out"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>Starting Out</code></h2>
<p>After downloading and unzipping the current <a href="https://github.com/NDDEV/archive/master.zip">repository</a>, read the user guide avaiable here (click download PDF):
<a href="https://www.overleaf.com/read/hsqjhvtbkgvj#/54715995/" rel="nofollow">https://www.overleaf.com/read/hsqjhvtbkgvj#/54715995/</a></p>
<p>The data for the following examples can be downloaded here:
<a href="https://drive.google.com/open?id=1LeQJFKaVCM2K59pKO9jDcB02yjTmJPmL" rel="nofollow">https://drive.google.com/open?id=1LeQJFKaVCM2K59pKO9jDcB02yjTmJPmL</a></p>
<pre><code>Example_1_NZ.m %&lt;- A simple mesh around South Island New Zealand that uses GSHHS shoreline. 
Example_2_NY.m %&lt;- A mesh around the Manhattan area that uses a DEM created from LiDAR data.  
Example_3_ECGC.m %&lt;- An example that builds a mesh for the western North Atlatnic with local nests around the mid-Atlantic.
Example_4_JBAY.m %&lt;- An extremely high-fidelity (15-m) mesh from LiDAR data around Jamaica Bay with CFL limiting.
Example_5_GBAY.m %&lt;- An example of the polyline/thalweg mesh size function along the Houston Ship Channel. 

</code></pre>
